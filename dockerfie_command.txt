FROM命令
新しいイメージ作成のベースとするイメージを指定する
FROM命令に指定するイメージをベースイメージと呼ぶ
Dockerfileの各命令による変更内容は、FROM命令に指定したイメージのレイヤ上に新しいレイヤを積み重ねていく
Dockerfileの命令は、FROM命令または、ARG命令から記述する必要がある

RUN
イメージ作成(ビルド)時にコンテナで実行するコマンドを指定する
パッケージのインストールなど、イメージに含めたいライブラリのインストールや、アプリ実行に必要なユーザの作成といった、
作成するイメージで事前に実行しておきたいコマンドを指定する際に使用する
ベースイメージに含まれるコマンドや、前のRUN命令でインストールしたコマンドであれば、RUN命令の引数に指定して実行することができる

CMD
コンテナを新規に作成して実行した際に、デフォルトで実行されるコマンドを指定する
docker runコマンドでイメージ名の後にコマンドを続けた場合は、CMD命令を上書きして、
dockr run コマンドの引数に指定したコマンド実行が優先されて実行される

ENTRYPOINT
CND命令と同様に、コンテナを新規に作成して実行した際にデフォルトで実行されるコマンドを指定する
CMD命令と異なる点としてdocker runコマンドの引数に実行するコマンドを指定した場合は、ENTRYPOINT命令に指定したコマンドを上書きするのではなく、
ENTRYPOINT命令のコマンドの引数にdocker runに指定したコマンドが渡されてコマンドが実行される
DockerfileにCMD命令とENTRYPOINT命令の両方が記述されていた場合は、コンテナ起動時にENTRYPOINT命令に指定したコマンドが実行され、
その引数にCMD命令コマンドが渡されることになる
このような動作から、ENTRYPOINT命令は常にコマンド起動時に実行したいコマンドがある場合に使用する

ENV
イメージに設定しておきたい環境変数を定義する
コンテナでアプリを実行するために、必要となる環境変数を、あらかじめ設定しておく際に使用される
ただし、パスワードや秘密鍵といった他人に公開してはいけない情報は、セキュリティの観点でイメージ内に直接含めてはいけない

ARG
イメージをビルドする際に一時的に使用する環境変数を設定する際に指定する
ENV命令とは異なり、イメージ上には環境変数は保持されず、ビルドの際に一時的に使用できる環境変数

USER
コンテナ内でコマンドを実行する際に、どのユーザーで実行するのかを指定する
USER命令を指定した以降のコマンドは、USER命令で指定したユーザーで実行される
USER命令を記述しなかった場合は、デフォルトでrootユーザーで実行されるが、セキュリティの観点で全てをrootユーザーで実行するのはくない
パッケージのインストールなど、root権限が必要な命令のみ、rootで行い、アプリの実行などは、アプリ実行用のユーザーを作成して使用する

WORKDIR
作業ディレクトリを移動する際に使用する

ADD
ローカル上のファイルをイメージにコピーする際に指定する
tar形式のアーカイブファイルの場合は、コピーした後に自動的に展開される
コピー元にURL指定することができ、URLを指定した場合は、URLからダウンロードしたファイルをイメージ内にコピーする動作になる

COPY
ADD命令同様ローカル上のファイルをイメージにコピーする際に指定する
tarアーカイブの展開やURLの指定はサポートしていない
